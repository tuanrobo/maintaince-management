Overview

Để đáp ứng các bộ phận kỹ thuật bán hàng tự động và giải quyết nhu cầu như: sửa chữa, nâng cấp, bảo trì về các thiết bị phần cứng của máy bán hàng tự động. Chúng tôi phân tích và cung cấp các giải pháp để tạo ra các công cụ hỗ trợ bộ phận kỹ thuật để giải quyết các vấn đề trên.

Expectation

Input system for spare parts management

Input parts list with quantity.

Each time Technician completes replace a part, he will confirm on Maintenance function on the machine.

And the machine push this information to server.

The server auto decreases quantity of this part on server and records it.

At the same time, ID machine/time done /ID of Technician to server.

Dashboard for monitoring

  1. Note jam
  2. Temperature inside the cabinet
  3. Door Status
  4. Slot broke
  5. Network status
  6. Maintenance Status(What machine need to clean Bill Validator clean Cooling system on next month...),
  7. System status

Notification alert needs to push for Technician group:

  1. Note Jam
  2. Door Status
  3. Lost connection between machine and server more than xx hours
  4. Temperature inside the cabinet above xxx

Record database for statistics

Record database about:

  1. Note jam
  2. Slot broke, any problem affect to sale.

This information statistic help us predict what are spare parts more broken. Record date-time after maintenance machine (After Clean Bill Validator or Clean Cooling system).

Setting for machines configuration on the server

Some parameters we can set on the server-side and sync to the machine without going to the machine and set on the machine. Example: Temperature limit, the domination of the Bill validator can accept (we already have this function on our server), the time turn on/turn off the light, and LCD screen.
